Odd Bitshttp://blog-beta.oddbit.com/One of these things is not like the othersMon, 05 Nov 2012 00:00:00 -0500Private /tmp directories in Fedorahttp://blog-beta.oddbit.com/posts/2012/11/05/fedora-private-tmp/<p>I ran into an odd problem the other day: I was testing out some configuration changes for a web application by dropping files into <code>/tmp</code> and pointing the application configuration at the appropriate directory. Everything worked out great when testing it by hand...but when starting up the <code>httpd</code> service, the application behaved as if it was unable to find any of the files in <code>/tmp</code>.</p> <p>My first assumption was that had simply missed something obvious like file permissions or that I had a typo in my configuration, but after repeated checks and lots of testing it was obvious that something else was going on.</p> <p>Grasping at straws I took a close look at the <code>systemd</code> service file for <code>httpd</code>, which looks like this:</p> <div class="highlight"><pre><span class="k">[Unit]</span> <span class="na">Description</span><span class="o">=</span><span class="s">The Apache HTTP Server (prefork MPM)</span> <span class="na">After</span><span class="o">=</span><span class="s">syslog.target network.target remote-fs.target nss-lookup.target</span> <span class="k">[Service]</span> <span class="na">Type</span><span class="o">=</span><span class="s">forking</span> <span class="na">PIDFile</span><span class="o">=</span><span class="s">/var/run/httpd/httpd.pid</span> <span class="na">EnvironmentFile</span><span class="o">=</span><span class="s">/etc/sysconfig/httpd</span> <span class="na">ExecStart</span><span class="o">=</span><span class="s">/usr/sbin/httpd $OPTIONS -k start</span> <span class="na">ExecReload</span><span class="o">=</span><span class="s">/usr/sbin/httpd $OPTIONS -t</span> <span class="na">ExecReload</span><span class="o">=</span><span class="s">/bin/kill -HUP $MAINPID</span> <span class="na">ExecStop</span><span class="o">=</span><span class="s">/usr/sbin/httpd $OPTIONS -k stop</span> <span class="na">PrivateTmp</span><span class="o">=</span><span class="s">true</span> <span class="k">[Install]</span> <span class="na">WantedBy</span><span class="o">=</span><span class="s">multi-user.target</span> </pre></div> <p>Browsing throught file the following line caught my eye:</p> <div class="highlight"><pre><span class="n">PrivateTmp</span><span class="o">=</span><span class="nb">true</span> </pre></div> <p>If you know about per-process namespaces in Linux, you're probably saying "Ah-ha!". If you <em>don't</em> know about per-process namespaces in Linux...you should, because this is the foundation for all sorts of things including Linux Containers (<a href="http://lxc.sourceforge.net/">LXC</a>). Here's some good introductory reading:</p> <ul> <li><a href="http://lxr.free-electrons.com/source/Documentation/unshare.txt">http://lxr.free-electrons.com/source/Documentation/unshare.txt</a></li> <li><a href="http://www.debian-administration.org/article/628/Per-Process_Namespaces">http://www.debian-administration.org/article/628/Per-Process_Namespaces</a></li> <li><a href="http://glandium.org/blog/?p=217">http://glandium.org/blog/?p=217</a></li> </ul> <p>In short, with this configuration in place, the service gets it's very own version of <code>/tmp</code> not shared with any other process. While the files I placed in <code>/tmp</code> were visible in <em>my</em> process, they didn't exist from the point of view of Apache.</p> <p>The fix in my case was to place the files somewhere other than <code>/tmp</code>. One could also disable the <code>PrivateTmp</code> setting, but it's generally turned on for reasons of security.</p> <p>The <code>PrivateTmp</code> option is documented in <a href="https://docs.fedoraproject.org/en-US/Fedora/17/html/Release_Notes/sect-Release_Notes-Changes_for_Sysadmin.html">Changes in Fedora for System Administrators</a>, and Dan Walsh discusses it briefly on <a href="http://danwalsh.livejournal.com/51459.html">his blog</a>.</p>Lars Kellogg-StedmanMon, 05 Nov 2012 00:00:00 -0500tag:blog-beta.oddbit.com,2012-11-05:posts/2012/11/05/fedora-private-tmp/fedorasystemd
