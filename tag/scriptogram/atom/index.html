Kellogg-Stedmantag:blog-beta.oddbit.com,2012-11-05:2012/11/05/posting-via-api/<p>Scriptogr.am has a <a href="http://scriptogr.am/dashboard/#api_documentation">very simple api</a> that allows one to <code>POST</code> and <code>DELETE</code> articles. <code>POST</code>ing an article will place it in the appropriate Dropbox directory and make it available on your blog all in one step.</p> <p>Here is how you could use this API via Curl:</p> <div class="highlight"><pre><span class="s-Atom">curl</span> <span class="s-Atom">\</span> <span class="o">-</span><span class="s-Atom">d</span> <span class="s-Atom">app_key=</span><span class="err">$</span><span class="nv">APP_KEY</span> <span class="s-Atom">\</span> <span class="o">-</span><span class="s-Atom">d</span> <span class="s-Atom">user_id=</span><span class="err">$</span><span class="nv">USER_ID</span> <span class="s-Atom">\</span> <span class="o">-</span><span class="s-Atom">d</span> <span class="s-Atom">name=</span><span class="s2">&quot;${title:-$1}&quot;</span> <span class="s-Atom">\</span> <span class="s-Atom">--data</span><span class="o">-</span><span class="s-Atom">urlencode</span> <span class="s-Atom">text@</span><span class="err">$</span><span class="s-Atom">tmpfile</span> <span class="s-Atom">\</span> <span class="s-Atom">\</span> <span class="nn">http</span><span class="p">:</span><span class="o">//</span><span class="s-Atom">scriptogr</span><span class="p">.</span><span class="s-Atom">am</span><span class="o">/</span><span class="s-Atom">api</span><span class="o">/</span><span class="s-Atom">article</span><span class="o">/</span><span class="s-Atom">post</span><span class="o">/</span> </pre></div> <p>This assumes that you've registered for an application key and that you have configured the value into <code>$APP_KEY</code> and your Scriptogr.am user id into <code>$USER_ID</code>.</p> <p>The <code>name</code> attribute is both the title of your article <em>and also</em> the filename (normalized and with a <code>.md</code> extension). Your article <em>must not</em> contain the metadata values that are allowed when you're editing files directly in the Dropbox directory.</p>
