PowerShellhttp://blog-beta.oddbit.com/posts/2012/11/04/powershell-wait-for-networking/<p>I've recently been exploring the world of Windows scripting, and I ran into a small problem: I was running a script at system startup, and the script was running before the network interface (which was using DHCP) was configured.</p> <p>There are a number of common solutions proposed to this problem:</p> <ul> <li> <p>Just wait for some period of time.</p> <p>This can work but it's ugly, and because it doesn't actually verify the network state it can result in things breaking if some problem prevents Windows from pulling a valid DHCP lease. - Use ping to verify the availability of some remote host.</p> <p>This works reasonably well if you have a known endpoint you can test, but it's hard to make a generic solution using this method.</p> </li> </ul> <p>What I really wanted to do was to have my script wait until a default gateway appeared on the system (which would indicate that Windows had successfully acquired a DHCP lease and had configured the interface).</p> <p>My first attempts involved traditional batch scripts (<code>.bat</code>) running some variant of <code>route print</code> and searching the output. This can work, but it's ugly, and I was certain there must be a better way. I spent some time learning about accessing network configuration information using PowerShell, and I came up with <a href="https://gist.github.com/4011808">the following code</a>:</p> <div class="highlight"><pre><span class="err">#</span> <span class="nb">Wait</span> <span class="nb">for</span> <span class="nx">a</span> <span class="nx">DHCP</span><span class="na">-enabled</span> <span class="nx">interface</span> <span class="k">to</span> <span class="nx">develop</span> <span class="err">#</span> <span class="nx">a</span> <span class="nb">default</span> <span class="nx">gateway.</span> <span class="err">#</span> <span class="err">#</span> <span class="nb">usage</span><span class="p">:</span> <span class="nb">wait</span><span class="na">-for-network</span> <span class="err">[</span> <span class="o">&lt;</span><span class="nx">tries</span><span class="o">&gt;</span> <span class="cp">]</span> function wait-for-network ($tries) { while (1) { # Get a list of DHCP-enabled interfaces that have a # non-$null DefaultIPGateway property. $x = gwmi -class Win32_NetworkAdapterConfiguration ` -filter DHCPEnabled=TRUE | where { $_.DefaultIPGateway -ne $null } # If there is (at least) one available, exit the loop. if ( ($x | measure).count -gt 0 ) { break } # If $tries &gt; 0 and we have tried $tries times without # success, throw an exception. if ( $tries -gt 0 -and $try++ -ge $tries ) { throw &quot;Network unavaiable after $try tries.&quot; } # Wait one second. start-sleep -s 1 } } </pre></div> <p>This uses various sort of filtering to get a list of DHCP-enabled interfaces that have a default gateway (the <code>DefaultIPGateway</code> attribute).
