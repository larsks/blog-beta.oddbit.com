Odd Bitshttp://blog-beta.oddbit.com/2011-07-26T00:00:00-04:00Fixing RPM with evil magic2011-07-26T00:00:00-04:00Lars Kellogg-Stedmantag:blog-beta.oddbit.com,2011-07-26:2011/07/26/fixing-rpm-with-evil-magic/<h1>Fixing rpmsign with evil magic</h1> <p>At my office we are developing a deployment mechanism for RPM packages. The general workflow looks like this:</p> <ul> <li>You build a source rpm on your own machine.</li> <li>You sign the rpm with your GPG key.</li> <li>You submit the source RPM to our buildserver.</li> <li>The buildserver validates your signature and then builds the package.</li> <li>The buildserver signs the package using a master signing key.</li> </ul> <p>The last step in that sequence represents a problem, because the <code>rpmsign</code> command will always, always prompt for a password and read the response from <code>/dev/tty</code>. This means that (a) you can't easily provide the password on stdin, and (b) you can't fix the problem using a passwordless key.</p> <p>Other people have <a href="http://jrmonk-techzine.blogspot.com/2010/06/how-to-sign-rpm-files-in-batch-mode.html">solved this problem using expect</a>, but I've opted for another solution which in some ways seems cleaner and in others seems like a terrible idea: function interposition using <code>LD_PRELOAD</code>.</p> <p>The <code>rpmsign</code> command prompts for (and reads) a password using the <code>getpass()</code> function call. If you look at the <code>getpass(3)</code> man page, you'll see that the function is defined like this:</p> <div class="highlight"><pre><span class="cp">#include &lt;unistd.h&gt;</span> <span class="kt">char</span> <span class="o">*</span><span class="nf">getpass</span><span class="p">(</span> <span class="k">const</span> <span class="kt">char</span> <span class="o">*</span><span class="n">prompt</span><span class="p">);</span> </pre></div> <p>So we start with the following short block of C code:</p> <div class="highlight"><pre><span class="cp">#include &lt;stdio.h&gt;</span> <span class="cp">#include &lt;unistd.h&gt;</span> <span class="kt">char</span> <span class="o">*</span><span class="nf">getpass</span><span class="p">(</span> <span class="k">const</span> <span class="kt">char</span> <span class="o">*</span><span class="n">prompt</span><span class="p">)</span> <span class="p">{</span> <span class="n">printf</span><span class="p">(</span><span class="s">&quot;I ATE YOUR PASSPHRASE.n&quot;</span><span class="p">);</span> <span class="k">return</span> <span class="s">&quot;&quot;</span><span class="p">;</span> <span class="p">}</span> </pre></div> <p>This -- when properly loaded -- will replace the standard C library <code>getpass()</code> function with our own version, which simply returns an empty string. This of course means we'll be using a passwordless key, but you could obviously have our replacement function return an actual password instead of an empty string. I would argue that by doing so you would not substantially increase the security of your solution.</p> <p>Next we create a shared library:</p> <div class="highlight"><pre><span class="err">$</span> <span class="n">cc</span> <span class="o">-</span><span class="n">fPIC</span> <span class="o">-</span><span class="n">g</span> <span class="o">-</span><span class="n">c</span> <span class="o">-</span><span class="n">o</span> <span class="n">getpass</span><span class="p">.</span><span class="n">o</span> <span class="n">getpass</span><span class="p">.</span><span class="n">c</span> <span class="err">$</span> <span class="n">ld</span> <span class="o">-</span><span class="n">shared</span> <span class="o">-</span><span class="n">o</span> <span class="n">getpass</span><span class="p">.</span><span class="n">so</span> <span class="n">getpass</span><span class="p">.</span><span class="n">o</span> </pre></div> <p>And now we perform our magic:</p> <div class="highlight"><pre><span class="err">$</span> <span class="n">LD_PRELOAD</span><span class="o">=</span><span class="err">$</span><span class="p">(</span><span class="n">pwd</span><span class="p">)</span><span class="o">/</span><span class="n">getpass</span><span class="p">.</span><span class="n">so</span> <span class="n">rpmsign</span> <span class="o">--</span><span class="n">addsign</span> <span class="n">some</span><span class="p">.</span><span class="n">src</span><span class="p">.</span><span class="n">rpm</span> <span class="n">I</span> <span class="n">ATE</span> <span class="n">YOUR</span> <span class="n">PASSPHRASE</span><span class="p">.</span> <span class="n">Pass</span> <span class="n">phrase</span> <span class="n">is</span> <span class="n">good</span><span class="p">.</span> </pre></div> <p>And <em>voila</em>! A solution for operating <code>rpmsign</code> in batch mode.</p>
