Since booting a single instance in OpenStack can result in the creation of several virtual interfaces, this results in a lot of:</p> <div class="highlight"><pre><span class="nx">May</span> <span class="mi">19</span> <span class="mi">11</span><span class="p">:</span><span class="mi">58</span><span class="p">:</span><span class="mi">24</span> <span class="nx">pk115wp</span><span class="na">-lkellogg</span> <span class="nx">NetworkManager</span><span class="err">[</span><span class="mi">1357</span><span class="cp">]</span>: <span class="nt">&lt;info&gt;</span> Activation (qvb512640bd-ee) starting connection &#39;Wired connection 2&#39; </pre></div> <p>You can disable this behavior by adding the following to <code>/etc/NetworkManager/NetworkManager.conf</code>:</p> <div class="highlight"><pre><span class="k">[main]</span> <span class="na">no-auto-default</span><span class="o">=</span><span class="s">*</span> </pre></div> <p>From <code>NetworkManager.conf(5)</code>:</p> <blockquote> <p>Comma-separated list of devices for which NetworkManager shouldn't create default wired connection (Auto eth0). By default, NetworkManager creates a temporary wired connection for any Ethernet device that is managed and doesn't have a connection configured. List a device in this option to inhibit creating the default connection for the device. May have the special value * to apply to all devices.</p> </blockquote> <h2>FirewallD</h2> <p><a href="https://fedoraproject.org/wiki/FirewallD">FirewallD</a> is the firewall manager recently introduced in Fedora (and soon to be appearing in RHEL 7).</p> <p>I start by creating a new zone named <code>openstack</code> by creating the file <code>/etc/firewalld/zones/openstack.xml</code> with the following content:</p> <div class="highlight"><pre><span class="cp">&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt;</span> <span class="nt">&lt;zone&gt;</span> <span class="nt">&lt;short&gt;</span>OpenStack<span class="nt">&lt;/short&gt;</span> <span class="nt">&lt;description&gt;</span>For OpenStack services<span class="nt">&lt;/description&gt;</span> <span class="nt">&lt;/zone&gt;</span> </pre></div> <p>After populating this file, you need to run <code>firewall-cmd --reload</code> to make the zone available. Note that if you're already running OpenStack this will hose any rules set up by Neutron or Nova, so you'll probably want to restart those services:</p> <div class="highlight"><pre><span class="c"># openstack-service restart nova neutron</span> </pre></div> <p>I then add <code>br-ex</code> to this zone, where <code>br-ex</code> is the OVS bridge my OpenStack environment uses for external connectivity:</p> <div class="highlight"><pre><span class="c"># echo ZONE=openstack &gt;&gt; /etc/sysconfig/network-scripts/ifcfg-br-ex</span> </pre></div> <p>I run a <code>dnsmasq</code> instance on my laptop to which I expect OpenStack instances to connect, so I need to add the <code>dns</code> service to this zone:</p> <div class="highlight"><pre><span class="c"># firewall-cmd --zone openstack --add-service dns</span> <span class="c"># firewall-cmd --zone openstack --add-service dns --permanent</span> </pre></div> <p>I'm running <code>firewall-cmd</code> twice here: the first time modifies the currently running configuration, while the second makes the change persistent across reboots.</p> <p>On my laptop, I handle external connectivity through NAT rather than placing floating ips on a "real" network. To make this work, I add my ethernet and wireless interfaces to the <code>external</code> zone, which already has ip masquerading enabled, by adding a <code>ZONE</code> directive to the appropriate interface configuration file:</p> <div class="highlight"><pre><span class="c"># echo ZONE=external &gt;&gt; /etc/sysconfig/network-scripts/ifcfg-em1</span> </pre></div> <p>After a reboot, things look like this:</p> <div class="highlight"><pre><span class="cp"># firewall-cmd --get-active-zones</span> <span class="n">openstack</span> <span class="nl">interfaces:</span> <span class="n">br</span><span class="o">-</span><span class="n">ex</span> <span class="n">external</span> <span class="nl">interfaces:</span> <span class="n">em1</span> <span class="n">public</span> <span class="nl">interfaces:</span> <span class="kt">int</span><span class="o">-</span><span class="n">br</span><span class="o">-</span><span class="n">ex</span> <span class="n">phy</span><span class="o">-</span><span class="n">br</span><span class="o">-</span><span class="n">ex</span> <span class="n">qvb58cc67ca</span><span class="o">-</span><span class="mo">06</span> <span class="n">qvo58cc67ca</span><span class="o">-</span><span class="mo">06</span> <span class="cp"># firewall-cmd --zone openstack --list-services</span> <span class="n">dns</span> </pre></div>Lars Kellogg-StedmanTue, 20 May 2014 00:00:00 -0400tag:blog-beta.oddbit.com,2014-05-20:2014/05/20/firewalld-and-openstack/openstacknetworkingfirewalldNetworkManager
