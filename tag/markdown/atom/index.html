Odd Bitshttp://blog-beta.oddbit.com/2012-11-06T00:00:00-05:00Converting HTML to Markdown2012-11-06T00:00:00-05:00Lars Kellogg-Stedmantag:blog-beta.oddbit.com,2012-11-06:2012/11/06/convert-html-to-markdown/<p>In order to import posts from <a href="http://blogger.com/">Blogger</a> into <a href="http://scriptogr.am/">Scriptogr.am</a> I needed to convert all the HTML formatting into Markdown. Thankfully there are a number of tools out there that can help with this task.</p> <ul> <li> <p><a href="http://markdownrules.com/">MarkdownRules</a>. This is an online service build around <a href="http://milianw.de/projects/markdownify/">Markdownify</a>. It's a slick site with a nice API, but the backend wasn't able to correctly render <code>&lt;pre&gt;</code> blocks. Since I'm often writing about code, my posts are filled with things like embedded XML and <code>#include &lt;stdio.h&gt;</code>, so this was a problem.</p> </li> <li> <p><a href="http://johnmacfarlane.net/pandoc/">Pandoc</a>. This is a general purpose tool that can convert between a variety of markup formats. Unfortunately, it <em>also</em> had similar problems with <code>&lt;pre&gt;</code> blocks.</p> </li> <li> <p><a href="https://github.com/aaronsw/html2text">html2text</a>. This a Python tool that converts HTML to Markdown. It seems to do a better job at handling the <code>&lt;pre&gt;</code> blocks, although it doesn't always get the indent level correct when the <code>&lt;pre&gt;</code> blocks are embedded in lists.</p> </li> </ul> <p>I ultimately ended up using <a href="https://github.com/aaronsw/html2text">html2text</a>, combined with a <a href="https://gist.github.com/4022537">simple script</a> to read the <a href="http://www.dataliberation.org/takeout-products/blogger">export from Blogger</a> and feed each document to the converter.</p>
