CrashPlan is a Java application, so this seemed like a possible solution. Unfortunately, Java under FreeBSD 8 seems to be a lost cause. I finally gave up and just installed Java under Linux.</p> <h2>Set up your Linux compatability environment</h2> <p>The simplest way to do this is to follow the instructions in the <a href="http://www.freebsd.org/doc/handbook/linuxemu-lbc-install.html">FreeBSD Handbook</a>. This will get you a Fedora 10 based Linux userspace, which should be more than sufficient. I'm using a CentOS 5.6 userspace, but for what we're doing it shouldn't matter, modulo some minor differences in paths.</p> <p>Note that Linux software running in this environment will have a modified view of your filesystem. In particular, /etc will map to /compat/linux/etc, and ZFS filesystems with non-default mountpoints seem to behave oddly (they are accessible, but not necessarily visible before you access them). This may require some workarounds in CrashPlan, depending on what you're trying to back up.</p> <h2>Install Java JRE</h2> <p>I installed a compatible Java environment from the CentOS package repository:</p> <div class="highlight"><pre><span class="cp"># chroot /compat/linux bash</span> <span class="n">bash</span><span class="o">-</span><span class="mf">3.2</span><span class="err">#</span> <span class="n">yum</span> <span class="n">install</span> <span class="n">java</span><span class="o">-</span><span class="mf">1.6.0</span><span class="o">-</span><span class="n">openjdk</span> <span class="n">bash</span><span class="o">-</span><span class="mf">3.2</span><span class="err">#</span> <span class="n">exit</span> </pre></div> <h1>Install CrashPlan</h1> <h2>Install the CrashPlan software</h2> <ul> <li> <p>Download <a href="http://www.crashplan.com/consumer/download.html?os=Linux">CrashPlan for Linux</a></p> </li> <li> <p>Unpack the archive (named something like CrashPlan_3.0.3_Linux.tgz)</p> </li> <li> <p>Change to the CrashPlan-install directory.</p> </li> <li> <p>Run the following commands:</p> <h1>export PATH=/compat/linux/usr/lib/jvm/jre-1.6.0-openjdk/bin:$PATH</h1> <h1>/compat/linux/bin/bash install.sh</h1> </li> <li> <p>Install CrashPlan into /usr/local. When prompted for where to locate init scripts ("What directory contains your SYSV init scripts?" and "What directory contains your runlevel init links?"), enter /tmp (because the installed init scripts aren't ideal for your FreeBSD environment -- we'll install our own later on).</p> </li> </ul> <h2>Fix Java</h2> <p>The Linux runtime provided by the FreeBSD Linux compatability layer does not include all of the features of recent Linux kernels. In particular, it is missing the epoll* syscalls, which will cause Java to die with a <em>Function not implemented</em> error. The workaround for this is documented in the <a href="http://wiki.freebsd.org/linux-kernel">linux-kernel</a> page on the <a href="http://wiki.freebsd.org/">FreeBSD wiki</a>:</p> <blockquote> <p>If you run an application in the linux java which wants to use the linux epoll functions (you should see "not implemented" messages in dmesg), you can start java with the argument -Djava.nio.channels.spi.SelectorProvider=sun.nio.ch.PollSelectorProvider</p> </blockquote> <h2>Install an rc script</h2> <p>Place the following script into /usr/local/etc/rc.d/crashplan:</p> <table
