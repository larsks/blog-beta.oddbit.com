Kellogg-Stedmantag:blog-beta.oddbit.com,2012-11-05:2012/11/05/posting-via-api/<p>Scriptogr.am has a <a href="http://scriptogr.am/dashboard/#api_documentation">very simple api</a> that allows one to <code>POST</code> and <code>DELETE</code> articles. <code>POST</code>ing an article will place it in the appropriate Dropbox directory and make it available on your blog all in one step.</p> <p>Here is how you could use this API via Curl:</p> <div class="highlight"><pre><span class="s-Atom">curl</span> <span class="s-Atom">\</span> <span class="o">-</span><span class="s-Atom">d</span> <span class="s-Atom">app_key=</span><span class="err">$</span><span class="nv">APP_KEY</span> <span class="s-Atom">\</span> <span class="o">-</span><span class="s-Atom">d</span> <span class="s-Atom">user_id=</span><span class="err">$</span><span class="nv">USER_ID</span> <span class="s-Atom">\</span> <span class="o">-</span><span class="s-Atom">d</span> <span class="s-Atom">name=</span><span class="s2">&quot;${title:-$1}&quot;</span> <span class="s-Atom">\</span> <span class="s-Atom">--data</span><span class="o">-</span><span class="s-Atom">urlencode</span> <span class="s-Atom">text@</span><span class="err">$</span><span class="s-Atom">tmpfile</span> <span class="s-Atom">\</span> <span class="s-Atom">\</span> <span class="nn">http</span><span class="p">:</span><span class="o">//</span><span class="s-Atom">scriptogr</span><span class="p">.</span><span class="s-Atom">am</span><span class="o">/</span><span class="s-Atom">api</span><span class="o">/</span><span class="s-Atom">article</span><span class="o">/</span><span class="s-Atom">post</span><span class="o">/</span> </pre></div> <p>This assumes that you've registered for an application key and that you have configured the value into <code>$APP_KEY</code> and your Scriptogr.am user id into <code>$USER_ID</code>.</p> <p>The <code>name</code> attribute is both the title of your article <em>and also</em> the filename (normalized and with a <code>.md</code> extension). Your article <em>must not</em> contain the metadata values that are allowed when you're editing files directly in the Dropbox directory.</p>Retrieving Blogger posts by post id2010-01-29T00:00:00-05:00Lars Kellogg-Stedmantag:blog-beta.oddbit.com,2010-01-29:2010/01/29/retrieving-blogger-posts-by-post-id/<p>I spent some time recently trying to figure out, using Google's <a href="http://code.google.com/apis/gdata/docs/2.0/basics.html">gdata</a> API, how to retrieve a post from a <a href="http://www.blogger.com/">Blogger</a> blog if I know corresponding post id. As far as I can tell there is no obvious way of doing this, at least not using the gdata.blogger.client api, but after much nashing of teeth I came up with the following solution.</p> <p>Given client, a <a href="http://gdata-python-client.googlecode.com/svn/trunk/pydocs/gdata.blogger.client.html">gdata.blogger.client</a> instance, and blog, a <a href="http://gdata-python-client.googlecode.com/svn/trunk/pydocs/gdata.blogger.data.html">gdata.blogger.data.Blog</a> instance, the following code will return a <a href="http://gdata-python-client.googlecode.com/svn/trunk/pydocs/gdata.blogger.data.html">gdata.blogger.data.BlogPost</a> instance:</p> <div class="highlight"><pre><span class="n">post</span> <span class="o">=</span> <span class="n">client</span><span class="p">.</span><span class="n">get_feed</span><span class="p">(</span><span class="n">blog</span><span class="p">.</span><span class="n">get_post_link</span><span class="p">().</span><span class="n">href</span> <span class="o">+</span> <span class="err">&#39;</span><span class="o">/%</span><span class="n">s</span><span class="err">&#39;</span> <span class="o">%</span> <span class="n">post_id</span><span class="p">,</span> <span class="n">auth_token</span><span class="o">=</span><span class="n">client</span><span class="p">.</span><span class="n">auth_token</span><span class="p">,</span> <span class="n">desired_class</span><span class="o">=</span><span class="n">gdata</span><span class="p">.</span><span class="n">blogger</span><span class="p">.</span><span class="n">data</span><span class="p">.</span><span class="n">BlogPost</span><span class="p">)</span> </pre></div> <p>I'm not sure if this is the canonical solution or not, but it appears to work for me.</p>
