From <a href="http://bit.ly/cBhU8N">http://bit.ly/cBhU8N</a>:</p> <blockquote> <p>NFS is built on ONC RPC (Sun RPC). NFS depends on RPC for authentication and identification of users. Most NFS deployments use an RPC authentication flavor called AUTH_SYS (originally called AUTH_UNIX, but renamed to AUTH_SYS).</p> <p>AUTH_SYS sends 3 important things:</p> <blockquote> <ul> <li>A 32 bit numeric user identifier (what you'd see in the UNIX /etc/passwd file)</li> <li>A 32 bit primary numeric group identifier (ditto)</li> <li>A variable length list of up to 16 32-bit numeric supplemental group identifiers (what'd you see in the /etc/group file)</li> </ul> </blockquote> </blockquote> <p>We ran into this today while diagnosing a weird permissions issue. Who knew?</p>Lars Kellogg-StedmanTue, 02 Feb 2010 00:00:00 -0500tag:localhost:8000,2010-02-02:posts/2010/02/02/nfs-and-the-16-group-limit/groupsnfs
