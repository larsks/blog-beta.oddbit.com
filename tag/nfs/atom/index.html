Odd Bitshttp://blog-beta.oddbit.com/2010-02-02T00:00:00-05:00NFS and the 16-group limit2010-02-02T00:00:00-05:00Lars Kellogg-Stedmantag:blog-beta.oddbit.com,2010-02-02:2010/02/02/nfs-and-16-group-limit/<p>I learned something new today: it appears that the underlying authorization mechanism used by NFS limits your group membership to 16 groups. From <a href="http://bit.ly/cBhU8N">http://bit.ly/cBhU8N</a>:</p> <blockquote> <p>NFS is built on ONC RPC (Sun RPC). NFS depends on RPC for authentication and identification of users. Most NFS deployments use an RPC authentication flavor called AUTH_SYS (originally called AUTH_UNIX, but renamed to AUTH_SYS).</p> <p>AUTH_SYS sends 3 important things:</p> <blockquote> <ul> <li>A 32 bit numeric user identifier (what you'd see in the UNIX /etc/passwd file)</li> <li>A 32 bit primary numeric group identifier (ditto)</li> <li>A variable length list of up to 16 32-bit numeric supplemental group identifiers (what'd you see in the /etc/group file)</li> </ul> </blockquote> </blockquote> <p>We ran into this today while diagnosing a weird permissions issue. Who knew?</p>
